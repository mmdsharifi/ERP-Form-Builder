# DESIGN.md — Design System & Tailwind Reference

This document maps out the visual styling guidelines, color schemes, and component styling conventions for the ERP Form Builder. It ensures that UI generated by LLMs matches the exact visual theme of the application without needing to scan components for styles.

## Overview

The ERP Form Builder uses a clean, modern, and accessible design system with a dual-mode styling configuration (Light and Dark) driven by Tailwind CSS. The interface is optimized for RTL (Right-to-Left) layouts.

## Colors

The project uses the following color palette:

| Element Role | Light Mode Tailwind Classes | Dark Mode Tailwind Classes |
| :--- | :--- | :--- |
| **Main Background** | `bg-[#f8fafc]` (slate-50) or `bg-white` | `bg-slate-900` or `bg-slate-950` |
| **Containers & Panels** | `bg-white` | `bg-slate-800` or `bg-slate-900` |
| **Standard Borders** | `border-gray-100` / `border-gray-200` | `border-slate-700/60` / `border-slate-800` |
| **Primary Brand (Accent)**| `indigo-500` / `indigo-600` | `indigo-400` / `indigo-500` |
| **Secondary Brand** | `teal-500` / `teal-600` | `teal-400` / `teal-500` |
| **Typography - Primary** | `text-gray-900` | `text-slate-100` |
| **Typography - Secondary**| `text-gray-600` / `text-gray-500` | `text-slate-300` / `text-slate-400` |
| **Typography - Muted** | `text-gray-400` | `text-slate-500` |

## Layout

Since the app runs in **RTL mode (`dir="rtl"`)**, standard horizontal classes must be handled with care:

- **DO NOT** use `ml-*` or `mr-*`. Instead, use `ms-*` (margin-start) and `me-*` (margin-end).
- **DO NOT** use `pl-*` or `pr-*`. Instead, use `ps-*` (padding-start) and `pe-*` (padding-end).
- **DO NOT** use absolute layout offsets like `left-*` or `right-*`. Instead, use `start-*` and `end-*`.
- **Flexbox Spacing**: Always include `space-x-reverse` when using horizontal gaps in flex containers (e.g., `flex space-x-2 space-x-reverse`).
- **Icons**: Chevron icons pointing to the right in LTR should point to the left in RTL. Check the view stack or navigation breadcrumbs and use appropriate transforms.

## Elevation & Depth

Always use these class mappings when styling interactive components:

- **Hover States**:
  - Border transitions: `hover:border-indigo-300 dark:hover:border-indigo-500`
  - Subtle ring hover: `hover:ring-2 hover:ring-indigo-100 dark:hover:ring-indigo-950/40`
  - Shadow growth: `hover:shadow-md transition-all`
- **Active / Selected State**:
  - Focus Ring: `border-indigo-500 shadow-[0_0_0_4px_rgba(99,102,241,0.15)] dark:border-indigo-400 dark:shadow-[0_0_0_4px_rgba(129,140,248,0.15)]`
  - Icon highlighted background: `bg-indigo-50 text-indigo-600 dark:bg-indigo-950/50 dark:text-indigo-400`
- **Draggable States**:
  - Draggable card: `cursor-grab active:cursor-grabbing`
  - Drop Target (Hovering): `border-dashed border-indigo-400 bg-indigo-50/50 dark:bg-indigo-950/20`

## Components

### A. Form Panels & Cards
```html
<div class="bg-white dark:bg-slate-800 border border-gray-100 dark:border-slate-700/60 rounded-xl shadow-sm p-4">
  <!-- Content -->
</div>
```

### B. Standard Input & Property Fields
For settings panels and property edits:
```html
<div class="space-y-1.5">
  <label class="block text-[11px] font-bold text-gray-400 dark:text-slate-500 uppercase tracking-wider">
    Field Label
  </label>
  <input 
    type="text" 
    class="w-full bg-white dark:bg-slate-800 border border-gray-200 dark:border-slate-700 rounded px-2.5 py-1.5 text-xs text-gray-700 dark:text-slate-200 focus:outline-none focus:border-gray-400 dark:focus:border-slate-500 transition-all shadow-sm"
  />
</div>
```

### E. Settings Panel Groupings
To minimize scrolling and keep a clean, flat style, avoid panel cards or background containers inside the settings. Use simple, flat dividers and prominent uppercase text for section headers:
```html
<div class="pt-3.5 border-t border-gray-100 dark:border-slate-880/60 space-y-2.5">
  <h4 class="text-xs font-bold text-gray-880 dark:text-slate-200">
    Section Name
  </h4>
  <!-- Flat settings inputs -->
</div>
```

### F. Active Drop Target Zones
Always provide responsive visual feedback when components are dragged over target drop zones:
- **Group Container (Form Canvas)**: Toggles a dashed indigo border, soft background highlight, slight scale increase, and focus ring:
  `className="border-2 border-dashed border-indigo-500 dark:border-indigo-400 bg-indigo-50/10 dark:bg-indigo-950/20 scale-[1.005] ring-2 ring-indigo-500/15 transition-all"`
- **Empty Field/Column Placeholder**: Highlights border and text color to match the drag theme:
  `className="border-indigo-500 text-indigo-600 bg-indigo-50/20 dark:border-indigo-400 dark:text-indigo-400 font-bold"`
- **Reordering Target Fields**: Highlights the field box with a solid color shift:
  `className="bg-indigo-50 dark:bg-indigo-950/20 border-indigo-400 dark:border-indigo-500 ring-2 ring-indigo-500/15"`
- **Grid Table Container**: Surrounds the grid table with a dashed indigo focus block:
  `className="ring-2 ring-indigo-500 bg-indigo-50/10 dark:bg-indigo-950/10 border-2 border-dashed border-indigo-500 dark:border-indigo-400 scale-[1.005] transition-all"`

### G. Helper Text & Tooltips
To save screen space, helper text (descriptions) is represented as a hoverable info icon tooltip next to the label:
```html
<label class="text-sm font-semibold text-gray-600 dark:text-slate-300 text-start flex items-center gap-1.5">
  Label Title
  <span class="relative group/tooltip flex items-center inline-block">
    <InfoIcon class="w-3.5 h-3.5 text-gray-400 hover:text-indigo-600" />
    <span class="absolute bottom-full mb-1.5 right-1/2 translate-x-1/2 w-48 p-2 bg-gray-900 text-white text-[10px] rounded-md shadow-xl opacity-0 invisible group-hover/tooltip:opacity-100 group-hover/tooltip:visible transition-all z-50">
      Tooltip description text
      <span class="absolute top-full right-1/2 translate-x-1/2 border-[5px] border-transparent border-t-gray-900" />
    </span>
  </span>
</label>
```
